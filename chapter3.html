Section 1

Introduction: The Heart of FOSS — GNU/Linux

Free and Open Source Software (FOSS) gives users four essential freedoms: run, study, redistribute, and improve. These freedoms apply to operating systems most fundamentally. Without a free OS, all other software is limited by the proprietary foundation beneath it.

An operating system (OS) manages your computer's hardware and provides the foundation for all other programs — translating keystrokes, saving files, allocating memory, and managing network connections.

💡 GNU/Linux: A Complete Free System
  • The Linux kernel (Linus Torvalds, 1991) — core that talks to hardware, GPL v2
  • The GNU tools (Richard Stallman, 1983) — essential utilities like GCC, Bash, Emacs
  • Thousands of free applications — browsers, office suites, all respecting your freedom
📌 GNU/Linux vs Linux
Most people say "Linux." The FSF says "GNU/Linux" to honor both contributing projects. Both names refer to the same OS.

Linux Distributions: Freedom of Choice

Unlike Windows or macOS, Linux comes in hundreds of distributions (distros). Each is a complete OS package: kernel, GNU tools, package manager, desktop environment, applications, and a community.

🟠 The Debian Family (founded 1993)

Distribution Based On Characteristics
Debian Independent The foundation. Extremely stable. Uses .deb packages and APT. Strict free software policy.
Ubuntu Debian Most popular desktop Linux. User-friendly, 6-month releases, LTS every 2 years. Backed by Canonical.
Linux Mint Ubuntu Beginner-friendly, Windows-like interface, multimedia codecs pre-installed.
Zorin OS Ubuntu Designed for Windows migrants. Multiple desktop layouts. Educational and Pro editions.
Pop!_OS Ubuntu Optimized for gaming and creative work. Excellent NVIDIA driver support.

Section 4

Installation: From Download to Boot

What is an ISO File?

An ISO file is a complete copy of a disc saved as a single file. Think of it as a perfect digital photograph of a DVD. It contains the complete OS, file system, boot information, and metadata.

Downloading & Verifying

cmd.exe
certutil -hashfile Zorin-OS-18-Core-64-bit.iso SHA256

Compare the output with the checksum on the official website. If they match, your download is perfect.

Creating Bootable USB Media

Tool Platform Notes
Rufus Windows Fast, reliable, recommended. Portable version available.
balenaEtcher All platforms Simple interface, cross-platform. Verifies written data.
Ventoy Windows, Linux Multi-ISO USB. Copy ISO files directly without rewriting.
dd Linux, macOS Direct disk writing. Powerful but dangerous if wrong device selected.
⚠️ Warning: All data on USB erased
Creating bootable USB erases all existing files on the drive. Backup first!

Section 5

Installation Methods Compared

🖥️

Virtual Machine

⭐ Recommended for This Course

Software simulates a complete computer within Windows/macOS. Linux runs as a program — fully functional but isolated.

✅ Advantages
  • Complete safety — mistakes contained to VM file
  • Windows installation untouched
  • Snapshots — save state and revert instantly
  • Run multiple distros simultaneously
  • Move VM files between computers
❌ Disadvantages
  • 10–20% slower than native
  • Shares RAM/CPU with host system
  • Graphics limitations for GPU work
  • VM files are large (20GB+)
🔧 Our choice: VirtualBox (GPL)
Free, cross-platform, easy to use, and a true FOSS tool itself.
⚡

Dual Boot

👷 Intermediate

Linux installed alongside Windows. GRUB boot loader lets you choose OS at startup.

✅ Advantages
  • Full native performance
  • Complete hardware access — full GPU, all RAM
  • Real-world driver experience
  • Can remove Windows later
❌ Disadvantages
  • Data loss risk — partitioning errors can erase files
  • Complex setup
  • Windows updates may overwrite GRUB
  • Must restart to switch OS
🐧

Single Boot

🔥 Expert

Linux is the only OS. Windows completely removed. Maximum performance and FOSS purity.

✅ Advantages
  • Maximum performance
  • Full disk utilization
  • Complete FOSS commitment
  • Smaller security attack surface
❌ Disadvantages
  • No fallback if Linux fails
  • Some Windows programs need alternatives
  • Too steep for beginners
Aspect Virtual Machine Dual Boot Single Boot
Safety for learning ✅ Excellent 🟡 Good ❌ Poor
Performance 🟡 Good (overhead) ✅ Excellent ✅ Excellent
Setup difficulty Easy Hard Medium
Risk to Windows None Low if careful Complete removal
Best for Learning, this course Daily Linux use Experts / servers

The Command Line Interface: Mastering the Terminal

The graphical user interface makes Linux accessible, but the command line makes it powerful. The terminal embodies the Unix philosophy: "Do one thing and do it well."

Open terminal in Zorin OS: Ctrl+Alt+T

The Prompt Explained

bash — student@zorin-os
student@zorin-os:~$
Component Meaning
student Current username
zorin-os Hostname (computer name)
~ Current directory (~ = home directory)
$ Regular user prompt
# Root/superuser prompt

Navigation Commands

pwd
Print Working Directory
$ pwd /home/student/Documents
ls
List Directory Contents
ls # basic listing ls -l # long format ls -la # include hidden files ls -lh # human-readable sizes ls -ltr # sorted by time
cd
Change Directory
cd /home # absolute path cd Documents # relative path cd .. # parent directory cd ~ # home directory cd - # previous directory

File and Directory Operations

mkdir
Make Directory
mkdir projects mkdir -p a/b/c # create nested dirs
touch
Create Empty File
touch file.txt touch a.txt b.txt c.txt
cp
Copy Files / Directories
cp file.txt backup.txt cp -r dir1 dir2 cp *.txt backup/
mv
Move or Rename
mv old.txt new.txt mv file.txt /dest/
rm ⚠️
Permanent delete — no trash!
rm file.txt # remove file rm -i file.txt # prompt first rm -r directory/ # remove folder # NEVER: rm -rf / — destroys system!
cat / nano / vim
View & Edit Files
cat file.txt # display file nano file.txt # simple editor vim file.txt # powerful editor

System Information & Networking

Command Purpose
whoami Current username
id User ID and group info
uname -a Kernel information
uptime System uptime and load
df -h Disk space usage (human-readable)
ip addr Network interface configuration
ping google.com Test internet connectivity

CLI vs GUI: A Comprehensive Comparison

⌨️

CLI

Text-based. Type commands. Powerful, precise, scriptable.

VS
🖱️

GUI

Visual. Windows, icons, menus. Accessible, discoverable, visual.

Aspect CLI GUI
Learning Curve Steep — memorize commands Gentle — intuitive metaphors
Speed Fast for experts, batch ops instant Fast for simple tasks
Reproducibility Commands scriptable, repeatable Sequences hard to reproduce
Resource Usage Minimal (~10MB RAM) Heavy (500MB–2GB RAM)
Remote Access Native via SSH, low bandwidth Requires VNC/RDP, high bandwidth
Automation Excellent — scripts, cron Limited macro recorders
Discoverability Poor — must know commands Good — menus show options

⌨️ Use CLI When:

  • Repetitive tasks (batch processing)
  • Remote servers via SSH
  • System administration
  • Writing automation scripts
  • Version control with git

🖱️ Use GUI When:

  • Learning a new application
  • Working with visual media
  • Browsing the web
  • Casual file management
  • Presenting to others
💡 Freedom Through Understanding
The CLI reveals the inner workings of your system. While GUIs hide complexity, the CLI exposes logic and structure. This transparency is fundamental to FOSS — you cannot truly understand or modify what you cannot see.
Section 8

Package Management: APT in Depth

APT (Advanced Package Tool) is the package manager for Debian-based distributions (Ubuntu, Zorin OS, Linux Mint). Unlike Windows/macOS hunting for installers online, Linux uses centralized repositories of tested, signed software.

Essential APT Commands

Update Package List
sudo apt update

Downloads latest package info. Does NOT upgrade installed software.

Upgrade Packages
sudo apt upgrade sudo apt full-upgrade

full-upgrade also handles dependency changes.

Install Software
sudo apt install gimp sudo apt install gimp inkscape vlc sudo apt install -y package

Use -y to skip confirmation prompts.

Remove Software
sudo apt remove package sudo apt purge package sudo apt autoremove

purge also removes config files. autoremove cleans orphans.

Search & Info
apt search keyword apt show package_name apt list --installed apt list --upgradeable
Fix Common Issues
sudo apt --fix-broken install sudo apt full-upgrade sudo apt autoremove && sudo apt clean
⚠️ Update vs Upgrade
apt update only updates the package list. It does NOT change any installed software. Always run this before installing or upgrading.

Repository Components

Component Description
main Officially supported free software
restricted Officially supported non-free firmware/drivers
universe Community-maintained free software
multiverse Non-free software (copyright restrictions)
